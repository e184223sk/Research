More than 3 years have passed since last update.ReSharperでC#を(比較的)null安全っぽくする方法のメモです。詳しくはJetBrainsのヘルプに記載されているので、そちらを参考にしてください。
ReSharper 2016.2 Help :: Annotations in Source Code
Value and Nullability Analysis以上NuGetでJetBarins.Annotationsを追加Visual Studioの右下の黒い丸を右クリックしてStart Solution Wide Analysisをクリックする時間かかるで！と確認メッセージが出るのでOKをクリックするVisual Studioの右下にXX warnings in XX filesとか出るようになる。
Value analysis modeをPessimisticにしていると[NotNull]属性がついていないプロパティや引数などをnullかどうかの確認を行わずに使用している個所はNullReferenceExceptionが発生する可能性があると判断されwarningとして計上される。nullかどうかの確認を事前に挟むことでwarningは解消される絶対にnullが入らない場合は[NotNull]属性を付けることでwarningが出なくなる属性に関してもヘルプにまとまっている。
ReSharper 2016.2 Help/Code Annotation Attributeswarningが出ていてもビルドはできるので注意は必要。
