しかし、当方、タグの通り、「電気系」の学生で「ガチ素人」であるため、先輩にまさかりを投げてもらいながら、覚えていく所存でございます。また、もし他に指摘点などございましたらぜひともよろしくおねがいします。

とりあえずできたはいいんですが、Androidの奴もコーディングしたいんですよね・・・・
いまはもう眠いのでGABの勉強して寝ます（決意）

追記！！

どうやら、最後の行に以下のメッセージがありました。

Could not install package 'Xamarin.Android.Support.Animated.Vector.Drawable 26.1.0.1'. You are trying to install this package into a project that targets 'MonoAndroid,Version=v7.1', but the package does not contain any assembly references or content files

その解決法は、
Androidのプロジェクトを開いて、右クリックして、オプションを選択すると、構成が開きます。

これで最新のバージョンに変換するとできてしまいました。

iOSの開発に切り替えて、「今までのはなんだったのか」と、案外早くに解決できてしまって、「今までのはなんだったのか」と感じていまいましたね。
なんとも・・・二段オチ

追記の追記

これで実行しても、エラーメッセージが・・・ということでこちらも調べました。

Exception while loading assemblies: 
System.IO.FileNotFoundException: 
Could not load assembly 'System.Configuration, Version=4.0.0.0, Culture=neutral, PublicKeyToken=b03f5f7f11d50a3a'. 
Perhaps it doesn't exist in the Mono for Android profile?

こちらのサイトさんに行きつきました。
http://d.hatena.ne.jp/omanuke-ekunamo/20180309/1520574867
そんなまさか・・・と思いながら、Androidのcsprojファイルの最初の＜propaty　group＞
に、

<AndroidSdkBuildToolsVersion>23.0.3</AndroidSdkBuildToolsVersion>

を挿入して、Nugetの追加から、v４を検索し、Xamarin.Android.Support.-v4をインストール。
そうすると、今度はHAXMがないぞと怒られながらも実行すると、死ぬほど遅い・・・

そんな時この情報を見つけたので、この回答の、hereをクリックし、zipを展開して、中のdmgでインストールすれば、

約１０秒で、エミュレーターが起動しました。（吐血）案外、調べると、コミュニティも英語しかなく、
C＃使いのプログラマの参考になれば、幸いです。

（注記）

他のサイト様の対処法を鵜呑みにし、他のV１３も同時に入れると、以下のメッセージがでてしまいました。おそらくパッケージが重複してしまったためでしょう。

Xamarin.Forms targets have been imported multiple times. Please check your project file and remove the duplicate import(s).
