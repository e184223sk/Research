[Unity]初心者向けコメントの書き方講座

C#VisualStudioUnity

この投稿は、Unity Advent Calendar 2017の12月17日の記事です。

この記事の対象となる方

・プログラミング＆Unity初心者（特に独学している方）
・アマチュアプログラマ
・個人開発者

※この記事は、プログラミングを独学している初心者で、「コメントってどう書けばいいの？」と思っている方向けの記事です。
プロのエンジニアとして企業にお勤めの方は、その現場の慣習や指示に従うようにしてください。

コメントとは？

プログラム内で以下のように記述することで、メモを残すことができます。
これをコメントと呼びます。

//（1行の場合）

/*
（複数行の場合）
*/

例えば、UnityでC#のプログラムを新規作成すると、自動的に以下のように入力されていますが、「// Use this for initialization」と「// Update is called once per frame」はコメントです。

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class NewBehaviourScript : MonoBehaviour {

    // Use this for initialization
    void Start () {

    }

    // Update is called once per frame
    void Update () {

    }
}

コメントは日本語で書いても構いません。

コメントを書くメリット

後で何をしていたかが分かりやすい

プログラミングでは、「3日後の自分は他人である」と言われます。
書いたときにはそのコードで何をしようとしていたかちゃんと頭の中にあったとしても、3日後には忘れてしまうことが多々ある（なのでコメントはちゃんと残しておきましょう）、という意味です。

他人が見たときに分かりやすい

ネット上のグループ（facebook、Slack等）やサービス（ココナラ等）、Q&Aサイト、知人等に質問する際、コメントが記入されていると、回答者にとってあなたがそのコードで何をしたかったのか分かりやすくなります。
（より適切に回答していただける可能性が高くなります。）

学習の助けになる

コメントを書こうと努力することによって、そのコードの意図や要約を頭の中で考えることになり、（全くコメントを書かないよりも）論理的思考やプログラミング力を磨くことができます。

最初の一歩

まずは上記にある、UnityでC#のプログラムを新規作成したときに自動で入力されるコメントの内容を、日本語にしてみましょう。

まずは「void Start」の説明を探すため、「void Start」で検索します。
（すでに「void Start」の意味を知っている方や、英訳が得意な方は、検索せずに自分の言葉でコメントを書いてもOKです。）

すると、以下の記事が見つかりました。
スクリプトの実行

この記事を参考に、コメントを書きます。

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class NewBehaviourScript : MonoBehaviour {

    // オブジェクト起動時に実行
    void Start () {

    }

    // フレーム毎に連続実行
    void Update () {

    }
}

「void ○○」が何を意味するかは、基本的な知識なのでいずれは記憶に定着し、書かなくても良くなると思います。
初心者のうちは上記のようにコメントを書いておくと、そのメソッドがいつ実行されるのか分かりやすくなります。

「そのコードで何をしたいのか」を書こう

以下は、コメントを全くしていないプログラムです。

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class NewBehaviourScript : MonoBehaviour {

    public GameObject hogeObject;

    public void ObjectSetActive()
    {
        hogeObject.SetActive(true);
    }
}

プログラミングに慣れた方が見れば、すぐに「ObjectSetActiveメソッドが呼び出された際、hogeObjectがアクティブになるプログラム」だと分かりますが、初心者のうちは一瞬に理解することは不可能です。

そこで、まずこのようにコメントを書きます。
（これより以下のコードでは、1～3行目の「using ○○」の部分は割愛します。）

public class NewBehaviourScript : MonoBehaviour {

    public GameObject hogeObject;

    // このメソッドが呼び出されたときに実行
    public void ObjectSetActive()
    {
        //hogeObjectをアクティブにする
        hogeObject.SetActive(true);
    }
}

これで、プログラムの目的がすぐに分かるようになりました。

パブリック変数の目的や種類を書こう

以下のように、パブリック変数で何をしたいのか（目的）や、何をその変数に指定すれば良いのか（種類）などを書いておくと、さらに分かりやすくなります。

public class NewBehaviourScript : MonoBehaviour {

    //アクティブにしたいゲームオブジェクト（キャラクター、ボタンなど）
    public GameObject hogeObject;

    // このメソッドが呼び出されたときに実行
    public void ObjectSetActive()
    {
        //hogeObjectをアクティブにする
        hogeObject.SetActive(true);
    }
}

インスペクターにパブリック関数に関するメモを表示させたいとき

ちなみにパブリック変数では、「[Header("コメントしたい内容")]」と書くことによって、インスペクターに注釈を表示させることができます。

public class NewBehaviourScript : MonoBehaviour {

    [Header("アクティブにしたいゲームオブジェクト（キャラクター、ボタンなど）")]
    public GameObject hogeObject;

    // このメソッドが呼び出されたときに実行
    public void ObjectSetActive()
    {
        //hogeObjectをアクティブにする
        hogeObject.SetActive(true);
    }
}

参考サイト：HeaderとSpaceを使ってInspectorを整理する

ソース（情報源）をメモしておこう

プログラミングの際、参考にしたサイトや書籍があれば、URLや書名、ページなどをコメントとして残しておくと、詳細を忘れてしまったときや、再度参照したいと思ったときに便利です。

public class NewBehaviourScript : MonoBehaviour {

    [Header("アクティブにしたいゲームオブジェクト（キャラクター、ボタンなど）")]
    public GameObject hogeObject;
    /* 参考サイト
    HeaderとSpaceを使ってInspectorを整理する
    http://tsubakit1.hateblo.jp/entry/2014/07/23/095513 */

    // このメソッドが呼び出されたときに実行
    public void ObjectSetActive()
    {
        //hogeObjectをアクティブにする
        hogeObject.SetActive(true);
        /* 参考サイト
        【Unity基礎】SetActiveを使ってゲームオブジェクトを表示・非表示にする方法
        http://freesworder.net/unity-setactive-true-false/ */
        /* 参考書籍
        UnityではじめるC# 基礎編　P153～155 */
    }
}

コメントが長くなりすぎてしまったとき

あまりコメントが長くなると、コード全体の見通しが悪くなってしまうことがあります。

そんなときは、「#region （タイトル）」と「#endregion」で囲むことによって、コードを折り畳むことができるようになります。
（コメント以外のコードも折り畳むことができます。）

・折り畳み前

・折り畳み後

    [Header("アクティブにしたいゲームオブジェクト（キャラクター、ボタンなど）")]
    public GameObject hogeObject;
    #region 参考サイト
    /* 参考サイト
    HeaderとSpaceを使ってInspectorを整理する

    public void ObjectSetActive()
    {
        //hogeObjectをアクティブにする
        hogeObject.SetActive(true);

コメントを考える際の心得

「正解」や「正しさ」にこだわらない

「コメントには何を書けばいいのだろうか」「この書き方でいいのかな」「こんな単純なことを書いてしまっていいのだろうか」と悩むこともあるかと思います。
しかし実際、コメントの書き方は人や現場によって様々です。
たった1つの「正解」というものはありません。

悩むことに多くの時間を使ってしまうよりも、気にせずどんどん書くようにしていった方が良いと思います。

「自分よりもプログラミングの知識を持たない人が見て分かるかどうか」で考える

冒頭の「3日後の自分は他人である」という格言にも繋がりますが、今覚えているコードや知識も、時間が経てば忘れてしまう可能性があります。

初心者の方ならば入門者の方が、中級者の方ならば初心者の方が見て分かるようなコメントを書くように心がけておくと、（主に自分が）後でとても助かります。

最後に

このように書いている私自身も、コメントを疎かにしてしまい、のちに後悔した経験が多々あります。
「（プロから見て）適切なコメント」を書くことは難しいですが、「自分があとから見たときに分かるコメント」であれば、書く努力をし続けていれば、初心者でも十分書けるようになります。
未来の自分を困らせないためにも、面倒でもなるべくコメントするようにお互い頑張っていきましょう。
