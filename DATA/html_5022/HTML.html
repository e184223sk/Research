受け取った日時文字列が許容時間の範囲内かどうかをチェックする

C#.NET

やりたいこと

クライアントから日時が文字列で渡されてくる。
ただし秒数の指定はない。

サーバと時刻がずれているかどうかをチェックしたい。
ただし、設定で指定した分数だけ許容分を設ける。

前提条件

  • 渡されてくる文字列のフォーマットは「yyyyMMddHHmm」とする
  • 文字列が所定のフォーマットに当てはまらない場合はNGを返す
  • 秒数考慮はせず、分までの値が一致していればOKとする
    • 例えば、10時15分(1015)が渡ってきて、許容分数が2であれば、10:13:00.000～10:17:99.999までの期間はOKとする
  • 許容分設定が0の場合もある

やってみよう

private bool isValid(string clientTime, long permitMinutes)
{
    // 日付パースできなければNG
    DateTime inputTime;
    if (!DateTime.TryParseExact(clientTime,"yyyyMMddHHmm", 
        System.Globalization.CultureInfo.InvariantCulture, 
        System.Globalization.DateTimeStyles.None,
        out inputTime))
    {
        return false;
    }

    // 現在日付 > 渡された日付の場合
    var now = DateTime.Now;
    if (now.Subtract(inputTime).Minutes > permitMinutes) { return false; }

    // 現在日付 < 渡された日付の場合
    var inputTimeMax = inputTime.AddSeconds(59).AddMilliseconds(999);
    if (inputTimeMax.Subtract(now).Minutes > permitMinutes) { return false; }

    return true;
}

現在日付 < 渡された日付の場合　に、59.999秒の猶予範囲を考慮してやらないと、現在時刻の秒数が減算されて1少ない数で判定されてしまう。

単純に差分の絶対値で判定するとうまくいかない。

上記判定だとミリ秒未満の考慮ができていない。
より直観的な判定処理をコメントいただきましたのでそちらもご参照ください。
